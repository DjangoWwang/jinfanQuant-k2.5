## 1. 基础概念与约定

### 1.1 符号定义

| 符号 | 含义 |
|------|------|
| NV_t | t日净值（复权净值） |
| R_i | 第i期收益率，R_i = NV_t / NV_{t-1} - 1 |
| R̄ | 收益率均值 |
| r_f | 无风险利率（年化，默认 2%） |
| r_m | 基准指数年化收益率 |
| n | 收益率观测个数 |
| T | 期初到期末的自然日天数 |
| σ | 年化波动率 |
| DD | 下行风险（年化） |

### 1.2 年化因子

采用**自然日天数法**计算年化因子，与火富牛保持一致：

```
年化因子 = 365 / 收益率平均间隔天数
```

其中"收益率平均间隔天数"= 相邻两个净值观测日之间的自然日天数的均值。

| 数据频率 | 典型平均间隔 | 典型年化因子 |
|----------|-------------|-------------|
| 日频 | ~1.4天 | ~252 |
| 周频 | ~7天 | ~52 |
| 月频 | ~30天 | ~12 |

**代码实现** (`metrics.py`):
```python
def _annualization_factor(nav_series):
    deltas = pd.Series(s.index).diff().dropna().dt.days
    avg_gap = float(deltas.mean())
    return 365.0 / avg_gap
```

### 1.3 无风险利率

- 默认值: **2%**（年化）
- 可配置: 系统设置中支持切换为固定值、十年期国债收益率、一年期定存利率或自定义值
- 日化换算: r_f_daily = (1 + r_f) ^ (1 / 年化因子) - 1

### 1.4 交易日历

- 数据源: A股交易日历（2016-2027年）
- 存储: `trading_calendar` 表
- 用途: 频率检测、区间对齐、回测模拟

## 2. 复权净值

复权净值表述分红再投资后的净值，用于计算基金真实收益率。

### 无份额拆分数据时

```
P_t = (1 + 净值涨跌幅) × P_{t-1}
    = ((L_t + S_{t-1} - L_{t-1}) / S_{t-1}) × P_{t-1}
```

其中:
- S_t = t日单位净值
- L_t = t日累计净值
- P_t = t日复权净值

### 有份额拆分数据时

```
P_t = ((S_t × T日拆分比例 + T日每份分红) / S_{t-1}) × P_{t-1}
```

若T日没拆分，拆分比例=1；若T日没分红，每份分红=0。

> **注**: 当前平台从火富牛获取的净值数据已为复权净值（viewv2接口返回的unit_nav），无需额外处理。

## 3. 收益率指标

### 3.1 期间收益率

```
r_period = NV_end / NV_start - 1
```

**代码**: `calc_return(nav_series, start_date, end_date)`

### 3.2 年化收益率（复利）

```
r_annual = (NV_end / NV_start) ^ (365 / T) - 1
```

其中 T = 期末净值日期 - 期初净值日期（自然日天数）。

**边界保护**: 当 `NV_end / NV_start - 1 <= -1`（即净值归零）时，返回 -1.0。

**代码**: `calc_annualized_return(nav_series, start_date, end_date)`

### 3.3 超额收益

支持两种计算方式:

| 方式 | 公式 |
|------|------|
| 除法超额（几何） | (1 + r_fund) / (1 + r_index) - 1 |
| 减法超额（算术） | r_fund - r_index |

默认采用**除法超额**（几何超额更精确）。

### 3.4 月度收益表

按自然月分组，每月收益 = 月末NAV / 上月末NAV - 1。

第一个月（无上月末基准时）使用月内首尾NAV计算。

## 4. 风险指标

### 4.1 年化波动率

```
σ = sqrt(∑(R_i - R̄)² / (n - 1)) × sqrt(年化因子)
```

其中:
- R_i = 第i期收益率
- R̄ = 收益率均值
- n = 收益率个数
- 使用样本标准差（ddof=1）

**代码**: `calc_annualized_volatility(nav_series, start_date, end_date)`

### 4.2 下行风险（下行偏差）

```
DD = sqrt(∑(min(R_i - r_f_daily, 0))² / (n - 1)) × sqrt(年化因子)
```

其中:
- R_i = 第i期收益率
- r_f_daily = 日化无风险利率 = (1 + r_f)^(1/年化因子) - 1
- min(R_i - r_f_daily, 0) 对**所有** n 个观测值计算（正超额裁剪为0）
- n = **全样本收益率个数**（非仅负数个数）
- 使用 ddof=1（除以 n - 1）

**注意**: 分母使用全样本 n-1，而非仅负超额收益的个数。这确保了当下行观测较少时不会低估风险。

**代码**: `calc_downside_deviation(nav_series, target_return)`

### 4.3 最大回撤

```
MaxDD = max((NV_peak - NV_trough) / NV_peak)
```

其中 NV_peak 出现在 NV_trough 之前。

返回值为**负数**（如 -0.15 表示 15% 回撤），同时返回峰值日期和谷值日期。

**回撤序列**: 对每个时间点 t:
```
DD_t = (NV_t - max(NV_1, ..., NV_t)) / max(NV_1, ..., NV_t)
```

**代码**: `calc_max_drawdown(nav_series)`, `calc_drawdown_series(nav_series)`

## 5. 风险调整指标

### 5.1 夏普比率 (Sharpe Ratio)

```
SR = (r_annual - r_f) / σ
```

其中:
- r_annual = 年化收益率
- r_f = 无风险利率（默认 2%）
- σ = 年化波动率

**代码**: `calc_sharpe_ratio(nav_series, risk_free_rate, start_date, end_date)`

### 5.2 索提诺比率 (Sortino Ratio)

```
SoR = (r_annual - r_f) / DD
```

其中:
- DD = 下行风险

**代码**: `calc_sortino_ratio(nav_series, risk_free_rate)`

### 5.3 卡玛比率 (Calmar Ratio)

```
CaR = r_annual / |MaxDD|
```

当最大回撤为0时，返回0。

**代码**: `calc_calmar_ratio(nav_series)`

### 5.4 信息比率 (Information Ratio)

```
IR = (r_annual - r_m) / TE
```

其中:
- r_m = 基准指数年化收益
- TE = 跟踪误差

### 5.5 VaR (95%置信)

```
VaR_95 = r_annual - 1.6449 × σ
```

## 6. 基准相关指标

### 6.1 Beta

```
β = Cov(R_i, R_m) / Var(R_m)
```

其中:
- R_i = 基金收益率序列
- R_m = 基准指数收益率序列
- 两个序列需对齐到共同日期

### 6.2 Alpha (Jensen's Alpha)

```
α = r_annual - (r_f + β × (r_m - r_f))
```

### 6.3 跟踪误差 (Tracking Error)

```
TD_i = R_i - R_m   (跟踪偏离度)
TE = sqrt(∑(TD_i - TD̄)² / (n - 1)) × sqrt(年化因子)
```

### 6.4 相关系数

```
ρ = Cov(R_i, R_m) / (σ_i × σ_m)
```

使用基金和基准在共同日期的收益率序列计算。

### 7.1 归一化净值

将净值序列重基到指定基准值（默认1.0），用于多基金对比图表:

```
NV_normalized_t = NV_t / NV_start × base
```

**代码**: `normalize_nav(nav_series, base, start_date, end_date)`

### 7.2 滚动指标

在每个净值点往前取固定滚动窗口，计算窗口内的指标（滚动收益、滚动年化、滚动夏普等）。计算方法与指标本身相同，差异仅在于取不同的滚动区间。

## 8. 回测引擎

### 8.1 引擎概览

回测引擎模拟一个由多只基金按目标权重构建的组合在历史区间内的表现。

**输入**:
- 基金权重 `{fund_id: weight}`（合计为1.0）
- 各基金NAV序列
- 回测区间 [start_date, end_date]
- 再平衡频率（日/周/月/季）
- 交易成本配置
- 频率对齐方式
- 历史模式（intersection / dynamic_entry / truncate）

**输出**:
- 组合NAV序列
- 回撤序列
- 绩效指标（使用上述公式计算）
- 月度收益表
- 动态进入日志（dynamic_entry模式）

### 8.2 模拟流程

```
[1] 获取全部基金NAV序列
[2] 频率对齐（见第9节）
[3] 计算收益率矩阵 returns_df
    - NAV先ffill(limit=30)处理停牌
    - pct_change()计算收益率
    - 去掉首行（pct_change产生的NaN）
[4] 过滤权重：仅保留returns_df中存在的基金，重新归一化
[5] 裁剪到回测窗口 [start_date, end_date]
[6] 确定再平衡日期（见8.4节）
[7] 逐日模拟循环:
    对每个交易日 t:
      a) 计算组合日收益: r_port = ∑(w_i × R_i_t)
      b) 更新组合NAV: NAV_t = NAV_{t-1} × (1 + r_port)
      c) 计算权重漂移: w_i_drift = w_i × (1 + R_i_t)
      d) 若 t 为再平衡日（且非首日）:
         - 归一化漂移权重
         - 计算换手率: turnover = ∑|w_drift_i - w_target_i|
         - 扣除交易成本: NAV_t *= (1 - cost(turnover))
         - 重置为目标权重
      e) 否则:
         - 归一化漂移权重作为下一日当前权重
[8] 计算绩效指标
[9] 返回结果
```

### 8.3 三种历史模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **intersection** | 取所有基金数据的共同日期交集 | 所有基金历史充足时 |
| **truncate** | 排除在start_date之前无数据的基金 | 容忍部分基金缺失 |
| **dynamic_entry** | 基金在其数据可用时动态进入组合 | 新成立基金较多时 |

### 8.4 再平衡日期

| 频率 | 触发条件 |
|------|----------|
| daily | 每个交易日（首日除外） |
| weekly | ISO周号变化时（含年份比较） |
| monthly | 月份变化时 |
| quarterly | 季度变化时 |

**关键**: 首日为建仓日，不计入再平衡日，不产生交易成本。

### 8.5 Dynamic Entry 模式

**进入时机**: 基金的第一个return出现在日期T，则该基金在T+1进入组合。

> 这避免了前视偏差（Point-in-Time）: T日的净值在T日收盘后才公布，不能在T日就使用T日的收益率做决策。

**进入后处理**:
1. 将新基金加入活跃集合
2. 按活跃基金的目标权重重新归一化
3. 如果不是首批进入（已有活跃基金），计算从旧权重到新目标的turnover，扣除交易成本
4. 记录进入日志

### 8.6 NaN防护

- NAV序列: 入库时验证 > 0，引擎层对小间隙做 ffill(limit=30)
- 收益率: 循环中遇到NaN视为0收益（停牌/未交易）
- 权重: 仅保留有效基金，重新归一化

## 9. 频率对齐

### 9.1 频率检测

基于相邻NAV观测日之间的**交易日数**的中位数判断:
- 中位数 <= 2: 日频
- 中位数 > 2: 周频

交易日数使用 bisect 二分查找计算（O(log n)）。

**代码**: `detect_frequency(nav_series, trading_days)`

### 9.2 对齐策略

| 场景 | downsample模式 | interpolate模式 |
|------|---------------|----------------|
| 全日频 | 不处理 | 不处理 |
| 全周频 | 统一到ISO week最后交易日 | 插值到日频网格 |
| 混合频率 | 日频降采样到周频 | 周频插值到日频 |

### 9.3 降采样到周频

将日频NAV按ISO week分组，取每周最后一个交易日的NAV值。

```
ISO Week 2026-W10:
  Mon 3/2: 1.052
  Tue 3/3: 1.055
  Wed 3/4: 1.048
  Thu 3/5: 1.051
  Fri 3/6: 1.053  <-- 取这个
```

### 9.4 周频规范化

当多只周频基金的观测日不同时（如A周三更新、B周五更新），统一对齐到每个ISO week的最后一个交易日:
- 如果基金在该周有观测值，取最近的
- 如果该周无观测值，forward-fill上周的NAV（不引入前视偏差）

### 9.5 插值到日频

对周频数据在交易日网格上做线性插值:

```python
s.reindex(daily_grid).interpolate(method="index")
```

> **注意**: 插值数据标记为估计值，用户需知晓其局限性。

## 10. 交易成本模型

### 10.1 成本类型

| 类型 | 说明 |
|------|------|
| 单边 (single) | 买入和卖出使用相同费率 |
| 双边 (double) | 买入和卖出分别配置费率 |

### 10.2 计算公式

**换手率 (Turnover)**:
```
turnover = ∑|w_drift_i - w_target_i|   (对所有基金求和)
```

其中 turnover 包含买入量和卖出量的总和。

**单边模式**:
```
cost = turnover × single_cost_bps / 10000
```

**双边模式**:
```
buy_amount  = ∑ max(w_target_i - w_drift_i, 0)
sell_amount = ∑ max(w_drift_i - w_target_i, 0)
cost = buy_amount × buy_bps/10000 + sell_amount × sell_bps/10000
```

### 10.3 成本扣除

```
NAV_after = NAV_before × (1 - cost)
```

### 10.4 不扣成本的场景

- **首日建仓**: 从现金到目标权重，不视为"调仓"
- **Dynamic entry首批基金**: 组合初始建仓时

### 10.5 默认参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| cost_type | single | 单边模式 |
| single_cost_bps | 10 | 10bps = 0.1% |
| buy_cost_bps | 0 | 双边模式买入费率 |
| sell_cost_bps | 0 | 双边模式卖出费率 |

## 11. 截面对齐与展示规则

> 这是区别于火富牛的核心设计。火富牛使用截止日但导致大量空值。
> 我们的原则：**净值领先的优先展示，落后的置底，用户可一键折叠未更新基金。**

### 11.1 截止日（截面日期）

用户通过日期选择器设定一个"截止日"，系统按净值新鲜度排序展示。

**展示规则**:
- 截止日当天有净值的基金：**正常展示**，按涨跌幅/净值等排序
- 截止日无净值的基金：**灰色置底**，涨跌幅显示"—"，标注最近净值日期
- 提供 **[折叠未更新]** 按钮：用户可一键隐藏无数据基金（个性化功能，非默认行为）

**日期选择器增强**:
- 选择日期时实时显示覆盖率："该日 32/42 只基金有净值"
- 提供 **[跳到最大覆盖日]** 一键跳转到最近的、使最多基金有数据的日期
- 周频基金密集出净值的日期（通常周五）高亮提示

```
┌─ 截止日: [◀] 2026-03-04 [▶] (32/42有数据)                      ┐
│  筛选: [全部(42)] [日频(30)] [周频(12)]   [折叠未更新] [跳到最大覆盖日] │
│                                                                 │
│  基金名称          频率   净值      涨跌幅    近一周    近一月   │
│  ─────────── 已更新 (32只) ──────────────────────────────────── │
│  紫杰黄金增强1号   日频   1.5941   -0.82%   -4.18%   +1.23%   │
│  鸣瑜300指增1号    日频   1.0289   -0.15%   -1.03%   +0.56%   │
│  量道1000A类       日频   1.1162   +0.08%   +0.24%   +2.11%   │
│  箐安期权稳健1号   周频   1.2350     --     +0.85%   +1.89%   │
│  ─────────── 未更新 (10只，灰色) ───────────────────────────── │
│  盛冠达量化C       周频   0.9585     —      —        +0.32%   │
│                                    最新: 2/28                   │
│  ...                                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 11.2 涨跌幅的定义（频率感知）

**涨跌幅 = 从上一个净值观测点到截止日的收益率。**

关键区别：
- **日频基金**: 涨跌幅 = 昨日→今日 = **1天收益**
- **周频基金**: 涨跌幅 = 上周→本周 = **1周收益**

**不同频率的涨跌幅不可直接比较！** 因此：

| 展示场景 | 处理方式 |
|---------|---------|
| 首页/基金池浏览 | 涨跌幅列正常显示，但周频基金在涨跌幅旁标注"周"字样 |
| 基金列表页 | 提供"统一口径"开关：开启后日频自动累计到周口径 |
| 基金比较页面 | 见11.3的严格对齐规则 |

### 11.2.1 不活跃基金自动识别

系统自动检测基金更新频率异常：
- **连续30天无新净值**: 自动标记"不活跃"标签，提醒用户关注
- **连续60天无新净值**: 建议用户剔除或归档
- 不活跃基金在列表中显示警告图标，用户可一键从基金池移除

### 11.3 基金比较页面的严格对齐规则

比较页面**不允许**展示不可比的数据。规则如下：

**规则1: 截止日必须所有基金都有净值**
```
选中: 基金A(3/4有数据), 基金B(3/4有数据), 基金C(最新3/2)
→ 提示: "基金C在3/4无净值数据。建议将截止日调整为3/2，或移除基金C。"
→ 提供快捷操作: [切到3/2] [移除基金C]
```

**规则2: 区间收益的起点也必须所有基金都有净值**
```
比较"近一周收益"，截止日=3/4:
  起点应为上周同一天: 2/25
  基金A(日频): 2/25有数据 ✓
  基金B(日频): 2/25有数据 ✓
  基金C(周频): 2/25无数据，最近为2/21(周五)
  → 提示: "周频基金C无法精确计算近一周收益。
     建议: [仅比较日频基金] 或 [切换到近一月收益]"
```

**规则3: 日频和周频只能在共同净值日比较**
```
日频基金 + 周频基金 同时比较:
  只有截止日恰好是周频基金的净值日时，才能比较
  例: 截止日=周五(周频基金有净值) → 可比较"近一周收益"
  例: 截止日=周三(周频基金无净值) → 折叠周频基金或提示用户切日期
```

**规则4: 一周 = 截止日所在周的实际交易日数**
```
正常周: 5个交易日
节假日周: 可能3-4个交易日
"近一周收益" = 截止日NAV / 上周同日NAV - 1
（不假设一周固定5天，而是从交易日历中取上周对应日）
```

### 11.4 频率筛选标签

顶部提供筛选标签，帮助用户快速过滤到可比基金集合：

```
[全部(42)] [日频(30)] [周频(12)]
```

- 点击"日频"：仅显示日频基金，所有区间收益精确可比
- 点击"周频"：仅显示周频基金，截止日自动对齐到最近的周五
- 点击"全部"：混合显示，但涨跌幅列标注频率，比较页面会提示不可比

### 11.5 不同页面的策略

| 页面 | 对齐严格度 | 无数据基金处理 | 涨跌幅含义 |
|------|-----------|--------------|-----------|
| **首页/仪表盘** | 宽松 | 显示最新净值+日期，不强制对齐 | 距上次更新的变化 |
| **基金列表页** | 中等 | 灰色置底，用户可一键折叠 | 标注频率(日/周)，可切统一口径 |
| **基金比较页** | 严格 | 强制对齐，不对齐则提示用户调整 | 统一区间，必须可比 |
| **回测引擎** | 最严格 | 由freq_align算法对齐，区分NAV date/publish date | 算法对齐后统一计算 |

### 11.6 截面日的应用场景

截面日不仅用于日常浏览，更重要的场景：

**市场大跌日复盘**:
```
截止日切到: 2026-02-03（假设A股大跌日）
→ 精确看到每只基金在那一天的表现
→ 筛选日频：看到当日涨跌幅
→ 筛选周频：看到包含该日的那一周表现
→ 对比各策略在极端行情下的抗跌能力
```

**周度总结**:
```
截止日切到: 上周五
→ 日频基金: 显示近一周收益（上上周五→上周五）
→ 周频基金: 恰好是其净值更新日，直接可比
→ 这是日频和周频唯一完美可比的时间点
```

### 12.2 月份回退规则

"近N月"的起始日期计算:
```
目标年月 = today - N个月
目标日 = min(today.day, 目标月最大天数)
```

例: today = 2026-01-31, 近3月:
- 目标月 = 2025年10月
- 10月最大天数 = 31
- 起始日 = 2025-10-31

### 12.3 交易日对齐

区间模板返回的是自然日，实际使用时需对齐到交易日:
- **起始日**: 对齐到最近的**后一个**交易日（不早于自然日）
- **结束日**: 对齐到最近的**前一个**交易日（不晚于自然日）

```python
def interval_dates_trading(preset, trading_days, reference_date):
    start, end = interval_dates(preset, reference_date)
    aligned_start = first_trading_day_on_or_after(start, trading_days)
    aligned_end = last_trading_day_on_or_before(end, trading_days)
    return (aligned_start, aligned_end)
```

### 12.4 区间收益与截面对齐的关系

预设区间的起止日期经过交易日对齐后，还需满足截面对齐规则（11.3）：
- 区间终点 = 截止日（用户选择的截面日期）
- 区间起点 = 按预设规则计算的日期，对齐到最近交易日
- 只有起点和终点都有净值的基金才展示该区间收益
- 不满足的基金在该列显示 `--` 或折叠

## 14. 回测中的净值对齐

回测引擎的净值对齐与前端展示不同，必须在算法层严格处理。

### 14.1 核心原则

回测中**不允许**任何近似或猜测：
- 参与回测的所有基金，在每个模拟时间点必须有真实（或降采样后的）NAV
- 不使用插值生成的"虚拟净值"参与收益计算
- 混频时强制降采样到最低频率（周频）
- **禁止未来函数**：回测中使用的净值必须在该时点已经发布

### 14.2 NAV date vs Publish date

私募基金净值存在两个日期概念：
- **NAV date（净值日期）**: 净值对应的估值日，如2026-03-05
- **Publish date（发布日期）**: 该净值实际对外公布的日期，如2026-03-08

在回测中，T日的决策只能使用 publish_date <= T 的净值数据。

```
例：基金A周五(3/5)的NAV在周二(3/8)才发布
  - 回测在3/5做再平衡时，不能使用3/5的NAV（尚未发布）
  - 回测在3/5可使用的最新NAV是3/4的（假设3/4已发布）
  - 回测在3/8才能使用3/5的NAV
```

**当前阶段**：由于火富牛API未提供publish_date，暂用nav_date + 固定延迟天数（默认1天）近似。
未来如有数据源提供publish_date，应直接使用。

### 14.3 降采样的精确定义

将日频基金降采样到周频时：
- 取每个ISO week中**最后一个有NAV的交易日**的净值
- 如果某周因节假日只有3个交易日，取第3天（而非强制周五）
- 周收益 = 本周末NAV / 上周末NAV - 1

### 14.4 周频基金的净值日差异

不同周频基金可能在一周内的不同日期更新：
- 基金A: 每周三更新
- 基金B: 每周五更新
- 回测对齐: 统一取每周最后一个交易日的可用NAV（forward-fill至该日）
- 这保证了所有基金在同一时间网格上有值

### 14.5 Forward-fill注意事项

- forward-fill产生的值应标记为"填充值"（非真实观测）
- 收益率计算只使用"真实→真实"区间，跳过"填充→填充"区间
- 波动率、最大回撤等指标只用真实净值点之间的收益率序列
- 长假期间（>5个交易日无真实值）的fill数据应特别标注
